# Итоговый отчет

## Чему я научилась на курсе

1. Я научилась строить приложения на базе паттерна MVC с использованием Java 17 и фреймворка Spring Boot 3. Поняла, как правильно разделять логику на слои (Controllers, Services, Repositories)
2. Освоила принципы проектирования реляционных баз данных (PostgreSQL). Научилась использовать Hibernate ORM и Spring Data JPA для связи объектной модели Java с таблицами базы данных, а также решила проблему каскадного сохранения сущностей (CascadeType)
3. Глубоко разобралась в механизмах аутентификации и авторизации. Реализовала бессессионную (stateless) защиту приложения с использованием JWT-токенов, которые передаются через безопасные HttpOnly Cookies. Настроила ролевую модель доступа (RBAC) с использованием аннотаций @PreAuthorize
4. Научилась проектировать REST API, документировать его по стандарту OpenAPI (через Swagger) и интегрировать сторонние сервисы (написание Telegram-бота для уведомлений).
5. Закрепила навыки работы с системой контроля версий Git (создание осмысленной истории коммитов), сборщиком проектов Maven и утилитами для тестирования API.

## Как это можно применить в будущем
1. Написания выпускной квалификационной работы (диплома), где потребуется надежная серверная часть.
2. Прохождения стажировок и собеседований на позицию Junior/Middle Java Developer, так как проект демонстрирует понимание enterprise-технологий. 
3. Построения безопасных микросервисов, так как реализованная мной JWT-авторизация и ролевая модель легко масштабируются на крупные коммерческие проекты.

## Источники, использованные при обучении
1. Официальная документация Spring - Spring Boot Reference Documentation & Spring Security Reference.
2. Стандарт RFC 7519 (JSON Web Token) для реализации системы токенов.
3. PostgreSQL Official Documentation и спецификации Hibernate ORM по работе со связями Many-To-Many.
4. OpenAPI Specification (OAS) 3.0 для настройки Swagger UI.
5. Платформы StackOverflow (для разбора узких архитектурных исключений, таких как TransientPropertyValueException), а также Telegram Bot API Manual.

## Заключение

Лабораторная работа выполнена в полном объеме. Разработанная кроссплатформенная система управления обновлениями полностью функциональна: она поддерживает CRUD-операции, выгрузку CSV-отчетов, загрузку конфигураций из файлов, генерацию тепловой карты и безопасное управление ролями.

Цели и задачи, поставленные в рамках курса, успешно достигнуты. Проект готов к деплою на виртуальный сервер и может служить реальным бэкендом для мобильных или десктопных приложений.
